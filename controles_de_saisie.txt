================================================================
  PAGE : CONNEXION (Login.fxml / LoginController.java)
================================================================

CHAMP : Adresse e-mail (emailField)
  - Declenchement : a chaque frappe (textProperty listener) + au clic sur "Se connecter"
  - Vide          : Erreur  → "⚠ Adresse e-mail obligatoire."
  - Format valide : format xxx@xxx.xxx accepte → champ passe au vert, erreur effacee
  - Format invalide: ne correspond pas au pattern → "⚠ Format invalide. Ex : vous@example.com"
  - Effet visuel  : bordure rouge (markError) ou verte (markValid) sur le champ

CHAMP : Mot de passe (passwordField)
  - Declenchement : a chaque frappe + au clic sur "Se connecter"
  - Vide          : Erreur  → "⚠ Mot de passe obligatoire."
  - Non vide      : Valide  → champ passe au vert, erreur effacee
  - Remarque      : pas de regles de complexite a la connexion (verification BCrypt cote serveur)

BOUTON : Effacer (handleClear)
  - Vide les deux champs
  - Remet les styles a zero (pas de rouge ni vert)
  - Efface les deux labels d'erreur
  - Efface le message de statut global


================================================================
  PAGE : INSCRIPTION (Inscription.fxml / RegisterController.java)
================================================================

CHAMP : CIN (cinField)
  - Declenchement : a chaque frappe + au clic sur "S'inscrire"
  - Valide        : exactement 8 chiffres → champ vert, erreur effacee
  - Vide          : "Le CIN est obligatoire."
  - Mauvais format: "Le CIN doit contenir exactement 8 chiffres."

CHAMP : Nom (nomField)
  - Declenchement : a chaque frappe + au clic sur "S'inscrire"
  - Non vide      : Valide → champ vert
  - Vide          : "Le nom est obligatoire."

CHAMP : Prenom (prenomField)
  - Declenchement : a chaque frappe + au clic sur "S'inscrire"
  - Non vide      : Valide → champ vert
  - Vide          : "Le prenom est obligatoire."

CHAMP : Date de naissance (dateNaissancePicker - DatePicker)
  - Declenchement : lors du changement de valeur + au clic sur "S'inscrire"
  - Selectionne   : Valide → champ vert
  - Non selectionne: "La date de naissance est obligatoire."

CHAMP : Telephone (telephoneField)
  - Declenchement : a chaque frappe + au clic sur "S'inscrire"
  - Vide          : "Le telephone est obligatoire."
  - 8 chiffres (ex: 54430709)      : Valide
  - +216 + 8 chiffres (ex: +21654430709) : Valide
  - Autre format  : "Format invalide. Exemple: 54430709 ou +21654430709."

CHAMP : Adresse e-mail (emailField)
  - Declenchement : a chaque frappe + au clic sur "S'inscrire"
  - Vide          : "L'adresse e-mail est obligatoire."
  - Format valide : xxx@xxx.xxx → Valide
  - Format invalide: "Adresse e-mail invalide."

CHAMP : Adresse (adresseField)
  - Declenchement : a chaque frappe + au clic sur "S'inscrire"
  - CHAMP OPTIONNEL : vide ou non, toujours considere valide
  - Si rempli     : champ passe au vert
  - Si vide       : style reset (ni rouge ni vert), pas de message d'erreur

CHAMP : Mot de passe (motDePasseField)
  - Declenchement : a chaque frappe + au clic sur "S'inscrire"
  - 8 caracteres minimum : Valide → champ vert
  - Moins de 8 caracteres : "Le mot de passe doit contenir au moins 8 caracteres."
  - Revalide aussi la confirmation a chaque changement

CHAMP : Confirmer le mot de passe (confirmerMotDePasseField)
  - Declenchement : a chaque frappe + au clic sur "S'inscrire"
  - Correspond au mot de passe ET mot de passe non vide : Valide
  - Ne correspond pas ou mot de passe vide : "Les mots de passe ne correspondent pas."

SOUMISSION FINALE (boutton S'inscrire) :
  - Tous les champs sont revalides
  - Si des erreurs, un message recapitulatif global est affiche en haut du formulaire
  - Si tout est valide, l'inscription est tentee (appel BDD), puis feedback succes/echec affiche

CHAMP : Nom (profileLastNameField)
  - Declenchement : a chaque frappe + au clic sur "Enregistrer"
  - Non vide      : Valide → champ vert
  - Vide          : "⚠ Nom obligatoire."

CHAMP : Telephone (profilePhoneField)
  - Declenchement : a chaque frappe + au clic sur "Enregistrer"

  - 8 chiffres (ex: 54430709)      : Valide
  - +216 + 8 chiffres (ex: +21654430709) : Valide

SOUMISSION FINALE (bouton Enregistrer) :
  - Tous les champs sont revalides
  - Si des erreurs, message recapitulatif global affiche
  - Si tout est valide, mise a jour BDD puis confirmation affichee


================================================================
  COMPORTEMENT VISUEL COMMUN AUX TROIS FORMULAIRES
================================================================

- Champ invalide  : bordure rouge (#ef4444 pour login.css, variante pour booking.css)
- Champ valide    : bordure verte (#22c55e)
- Label d'erreur  : texte rouge clair (#fca5a5 login / #f87171 compte), 11px, gras
                    classe CSS : .field-error
- Trigger         : validation en temps reel (listener sur textProperty / valueProperty)
                    + validation complete au moment de la soumission
- Reset           : bouton "Effacer" (login) ou rechargement de la page remet tout a zero
